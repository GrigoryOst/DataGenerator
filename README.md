Полноценный конвейер обработки данных в реальном времени:  
генерация показаний датчиков → передача через Apache Kafka → обработка с помощью Spark Structured Streaming → сохранение в HDFS (или локальную ФС).  

Обработка в реальном времени – данные обрабатываются по мере поступления (микропакетами).  
Отказоустойчивость – используются checkpoint'ы Spark для восстановления после сбоев.  
Лёгкое развёртывание – полная инфраструктура поднимается одной командой docker-compose up.  
Гибкая настройка – можно менять логику генерации, преобразования, формат хранения (JSON, Parquet).

Схема:  
DataGenerator (Spark) ────▶ Kafka (broker) ────▶ DataTransformer (Spark) ────▶ HDFS / Локальные файлы

1. DataGenerator – Spark-приложение, генерирующее поток случайных показаний (sensorId, temperature, timestamp) и отправляющее их в Kafka;
2. Kafka – брокер сообщений, хранит входящий поток (топик sensor-data);
3. DataTransformer – Spark Structured Streaming читает из Kafka, парсит данные, округляет температуру и сохраняет результат (по умолчанию в локальную ФС, опционально в HDFS);
4. HDFS – распределённое хранилище для долгосрочного хранения обработанных данных.

Cтек:  
Docker Compose;  
Apache Kafka (режим KRaft);  
Apache Spark (Structured Streaming);  
Hadoop HDFS (или локальная файловая система);  
Java 8;  
Maven.
